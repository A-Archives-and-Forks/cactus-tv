# v0.8.5

- Streamflow 改为固定前方 600 秒滚动窗口，避免长影片触发超大预取。
- 首帧稳定后才启动预取，并加入服务端任务锁和自适应批量，减少播放竞争与重复下载。
- 性能面板可关闭，关闭后显示轻量状态胶囊；状态面板会自动收起。
- 搜索元数据失败不再拖垮正常搜索，部分失败结果不再长时间缓存。
- 统一前端资源版本，避免浏览器混用旧播放器文件。

## 0.8.4 Streamflow 代理授权与可见状态修复

- 修复数据源已开启代理，但实际媒体 CDN 不在静态白名单时，播放器仍直连且 Streamflow 面板完全隐藏的问题。
- 新增基于 `ADMIN_TOKEN` 的 24 小时短期媒体凭证，仅授权当前私有站点为指定数据源代理媒体请求。
- HLS 主清单、子清单、音轨、密钥、初始化分片和视频分片都会继承该凭证。
- CactusStreamflow 面板始终显示：运行中、等待信息、已关闭、未启动或具体错误原因，不再静默消失。
- 保留媒体域名白名单；没有 `ADMIN_TOKEN` 时仍按原白名单规则工作。
- 无需新增 Cloudflare 绑定、D1 表、R2、Queue 或 Worker。

## 0.8.2 CactusStreamflow 全程 600 秒滚动缓存

- 移除“观看超过总时长三分之一才启动”的门槛。
- 播放器识别到 HLS 和有效时长后立即建立缓存窗口，从开头播放或断点续播都生效。
- 始终维持当前位置前方至少 600 秒缓存；接近片尾时缓存到片尾。
- 播放、暂停、恢复和拖动进度后都会按当前位置重新计算滚动窗口。
- 性能面板不再显示“等待 33%”，改为“等待视频信息 / 准备预取 / 流式预取中”。
- Cloudflare 配置不变，仍不需要 R2、Queue、独立 Worker或新增 D1 表。

## 0.8.1 CactusStreamflow 600 秒滚动缓存

- 新增缓存游标：每批从上次缓存结束位置继续，不再反复扫描前几十个分片。
- 观看超过三分之一后，目标窗口至少覆盖未来 600 秒；若剩余时长的一半更长，则继续按更长区间滚动预取。
- 播放中约每 10 秒续一批；暂停后每 7 秒续一批，达到目标或片尾后自动停止。
- 每批最多处理 9 个 HLS 对象，控制在 Cloudflare Free 每次请求 50 次 Cache API/子请求额度内。
- 新增播放器性能面板：显示已覆盖秒数、新增分片、复用分片和 hls.js 实际播放 HIT/MISS 命中率。
- 新增边缘状态记录与 20 秒并发保护，避免同一影片重复启动重叠预取批次。
- 仍然只需原有 Pages + D1；不需要 R2、Queue、额外 Worker 或信用卡。

## 0.8.0 CactusStreamflow Cache API

- 删除 R2、Queue、独立缓存 Worker 和 `0003_streamflow.sql` 依赖。
- CactusStreamflow 改用 Cloudflare Cache API，不需要信用卡或额外 Cloudflare 资源。
- `/api/stream` 对 HLS 分片先查边缘缓存，未命中时回源并异步写入缓存。
- 观看超过总时长三分之一后，每约 30 秒预取一小批后续 HLS 对象。
- 页面隐藏或退出时最多再尝试预取 12 个对象，但不承诺完整下载剩余区间。
- 支持主清单、多清晰度、AES-128、fMP4 初始化分片和 Byte Range 的稳定缓存键。
- 设置页改为显示 Cache API 模式，并通过切换缓存代数实现逻辑清空。
- 缓存是当前数据中心的临时缓存，容量不可统计，也不保证永久保存或跨地区命中。

## v0.5.0 Turbo

### 极致流畅度

- 首页由一次性渲染整页卡片改为首屏小批量渲染，接近视口或横向滚动时再追加。
- 搜索结果“显示更多”只追加新卡片，不再销毁并重建已经显示的节点。
- 图片改为 IntersectionObserver 预加载，低端设备使用更保守的预取距离。
- 长剧集列表按批次创建按钮，避免数百集同时占用 DOM 和事件资源。
- 自动识别省流量、低内存、低核心数和减少动态效果设备，关闭高耗 GPU 模糊、阴影和动画。
- 平板横屏海报取消每张卡片一层实时模糊背景，保留完整人像且降低合成开销。

### 加载速度

- 首页数据与站点健康检查并行加载，不再串行等待。
- 播放器界面和播放引擎改为真正按需加载，首页不解析播放器模块和 hls.js。
- 首屏脚本使用 modulepreload，首页、详情和搜索的浏览器缓存时间延长。
- 豆瓣海报优先请求适合卡片的中等尺寸，并直接尝试列表海报；资料页查询只在原图失效时回退。
- 海报浏览器缓存提高到 1 天，Cloudflare 边缘缓存提高到 7 天，并启用长期 stale-while-revalidate。
- 播放器时间轴更新合并到 requestAnimationFrame，减少频繁布局与绘制。
- 低端设备缩小 HLS 前后缓冲区，降低内存峰值和后台解码压力。

### 管理后台

- `ADMIN_TOKEN` 最低长度由 16 个字符调整为 8 个字符，前端和服务端规则保持一致。
- 仍建议使用随机字母、数字和符号组合；8 位只代表最低可用长度。

### 部署

- 不新增数据库迁移、Binding 或环境变量。
- 覆盖全部文件并提交到原 GitHub 仓库即可自动部署。

## v0.4.0

### 播放与容灾

- 播放失败后自动尝试同一数据源的其他线路。
- 同源线路全部失败后，按接口延迟和本机历史成功率尝试其他数据源。
- 自动换源时保留当前播放进度、集数和字幕入口。
- 播放器增加上一集、下一集和手动切换备用线路。
- 播放结束后显示 5 秒倒计时并自动进入下一集，可在设置中关闭。
- 当前使用的数据源和线路会显示在播放器下方。

### 页面与路由

- 增加首页分类入口。
- 搜索结果增加类型、年份和片源筛选。
- 搜索结果每次先渲染 24 条，可继续加载，降低手机和平板的瞬时渲染压力。
- 增加 `/detail/...`、`/watch/...`、`/category/...`、`/search`、`/favorites` 和 `/history` 直达路由。
- 刷新播放页或详情页后可以恢复对应页面，不再只能回到首页。

### 性能

- 搜索 API 使用 Cloudflare Cache API 缓存结果。
- 上游数据源搜索限制为最多 5 路并发，避免大量数据源同时请求造成抖动。
- 浏览器搜索缓存由 30 秒延长至 2 分钟。
- 继续保留请求去重、超时取消、图片懒加载和播放器资源释放。

### 部署

- 不新增数据库迁移。
- 不新增 Cloudflare Binding 或环境变量。
- 已连接 GitHub 的 Cloudflare Pages 项目只需提交代码，等待自动部署。

## 0.7.0 CactusStreamflow

- 新增 CactusStreamflow 仙人掌流式缓存。
- 观看不足总时长三分之一时不创建云端缓存。
- 达到阈值后缓存当前位置到剩余时长一半之间的 HLS 分片，单会话上限约 950 MiB。
- Pages 每 30 秒写入 D1 心跳，并通过延迟 Queue 消息处理浏览器直接退出的情况。
- 已有缓存时支持边看边滚动扩展，后台删除已经看完的视频分片。
- `/api/stream` 优先读取私有 R2，未命中时无缝回源。
- 支持 HLS 主清单、多清晰度、独立音轨、AES-128、fMP4 初始化分片和 Byte Range。
- 设置页新增缓存状态、占用、任务错误和清空所有 R2 缓存。
- 新增 `migrations/0003_streamflow.sql` 与独立 `streamflow-worker/`。
